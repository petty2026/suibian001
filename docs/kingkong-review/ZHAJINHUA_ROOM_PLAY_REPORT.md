## 流程概览

```
炸金花 Join 页 → 输入房号 / 直链进 detail
  → Cocos 牌桌（等待 / 入座）
    → 点击空座 → 「带入」弹窗
      → 确认带入 → 已座等待 / 「离座」
        → 满员开局 → 发牌 / 下注 / 比牌（两房间均未测到）
```

**结论：** 两房间均成功进桌；**501813** 见玩家 Arie 已座、**984332** 同样仅 1 人在座（Arie 21.00）。自动化与真机均**未能凑齐人数完成一整局**。以下合并两房间及真机 984332（已座 + 「离座」）发现。

## 合并问题清单

### 产品问题

1. 带入默认 **¥0** 低于最小 **¥20**，用户不知须手动调高  
2. **5 分钟未开局自动解散** 进房/点座后才提示，Join 页无预告  
3. **局数 0/8** 与 **回合 0/10** 双计数；**加注封顶**规则仅印在牌桌中央  
4. 等待文案「**恭候您的加入**」与已有玩家 Arie 同时存在，像空房  
5. 房号键盘输入 **501813 / 984332** 均未能稳定自动进房  
6. 单人/少人**无法开局**，无「还需 N 人才能开始」  
7. **进房 ≠ 入座**，须二次点空座 + 带入  
8. **KKC / ¥** 双余额、帐号 **`%Jimmy`** 贯穿流程  

### 交互问题

1. 带入 **¥0** 时「确定」仍为金色可点，与最小 ¥20 冲突  
2. **5 分钟解散** 提示叠在 **带入** 弹窗上，双层 modal  
3. 入座路径长，无「**一键最小带入**」  
4. **H5 关闭 tooltip** 遮挡 Cocos 带入界面  
5. 点头像 → 资料卡信息过少（ID / 胜率），无房主标识  
6. **「离座」** 无二次确认（984332 真机已座态）  
7. 分享 / 菜单 / 统计 / **表情** 点击反馈弱  
8. 退出语义三套：H5 关闭、右上箭头、菜单  

### 视觉问题

1. 带入 / 提示 / 资料卡 **共用弹窗模板**，严重级别不分  
2. 大号金色 **¥ 0** 比最小限额更醒目  
3. 牌桌中央信息过密（Logo / 房号 / 底分 / 局数 / 回合 / **加注封顶**）  
4. **KING KONG** 英文与中文规则混排；Cocos 字体非 SF  
5. 等待横幅遮挡规则文字  
6. 空座椅图标无「点击入座」；**离座** 与空座图标视觉层级混乱
