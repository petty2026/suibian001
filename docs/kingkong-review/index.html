二、核心问题与建议

P0 首屏性能

问题数据建议
JS 过大主包 1.17MB + chunk 616KB路由级拆包，首屏仅加载 game 模块
图片过多75+ 张图片请求lazy-load + WebP + 首屏限 12 张
FCP 慢4.19spreconnect API、延迟 CNZZ 统计

P0 信息架构

  • Login、Download APP、滚动公告、PWA Install、悬浮客服/活动 — 全部挤在首屏
  • 建议：只保留 1 个主 CTA；Download/PWA 延迟或非阻塞展示
  • Classic Mode 切换后视觉反馈弱，用户难以感知模式变化

P1 无障碍 & 规范

检查项现状目标
图片 alt0/28100% 覆盖
点击热区5 处 < 44px全部 ≥ 44px

P1 国际化

  • UI 英文 + Banner 中文混排，品牌感不统一
  • 公告栏英文在窄屏被截断
  • PWA 安装文案写 “install to desktop”，移动端应显示「添加到主屏幕」

  • ✅ 内联 Boot Skeleton — 减少白屏
